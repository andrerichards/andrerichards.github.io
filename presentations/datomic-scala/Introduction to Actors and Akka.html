Actors

Carl Hewitt 1973

  • Lightweight computational entities
  • Purely reactive
  • Communicate via async message passing
  • Made popular by Erlang
  • A form of Object Oriented Programming

Reactive

  • Mailbox
  • Reply to messages
  • Spawn new actor
  • Change behaviour

Akka

  • Toolkit for concurrent and distributed computing
  • JVM implementation of Actor model
  • Scala, Java API
  • Built-in fault tolerance
  • Scales up and out

Some prerequisites

  • Case classes for lightweight immutable objects
  • Pattern matching for switching on messages
  • Symbol-like operators (!, ?)

Example: Hello World

Behind the scenes

Vastly simplified

  • Mailboxes (Queues)
  • Dispatchers (thread pools)
  • Java Memory Model

Key concepts

  • You don't call methods, you send messages
  • Isolated mutability approach to concurrency
  • All actors are hidden behind a reference
  • Location transparency
  • Proxying, load balancing, etc...

Example: Actors in Holywood

Actor hierarchies

  • Actors can create children (and so on recursively)
  • Parents supervise ther children
  • All children may be equal (or not) in the face of errors

Example: dodgy math

What I use Akka for

Scala Actors for Media Processing

  • Large scale PVR
  • Near-live video editing
  • Re-streaming
  • Video archiving
  • High bit-rate feeds

High level picture

  • Pipeline of single-purpose media processing actors
  • Messages flow from actors to actors
    • UDP packets, RTP packets
    • Timecode, closed caption
    • Audio and video frames

A lot more

  • Using Actors with futures
  • Low level IO with Akka-io
  • Async HTTP REST Web Services with Spray

Pros

  • Scales very well up and out
  • Fits well with messaging oriented domain
  • Async/non blocking can be quite efficient

Cons

  • Lack of type safety
  • Actor hammer / concurrency nail
  • Async can be difficult to reason about
